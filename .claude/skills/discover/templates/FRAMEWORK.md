# R&D Framework

> "I don't build products: I build understanding. Artifacts are instruments of inquiry."

## Foundations

This framework integrates four methodological traditions:

### Grounded Theory (Glaser & Strauss)
Theory emerges from data — not the other way around. Categories are discovered through open coding, not imposed. The Glaser test (Fit, Work, Relevance, Grab) validates when a pattern is ready to become theory.

### Action Research (Lewin)
Knowledge through cycles: **Plan → Act → Observe → Reflect**. Each cycle produces both understanding and action. The reflection phase decides: continue, pause, or close.

### Design Science Research (Simon)
Creating artifacts generates knowledge. Building something reveals what you didn't know. The artifact is not the goal — the understanding gained through building is.

### Zettelkasten (Luhmann)
Knowledge emerges from connections. Individual notes (lines, theories) gain meaning through their relationships. The map of connections IS the knowledge system.

## Core Concepts

### Lines of Inquiry
A line is a sustained investigation driven by a central question. Lines have phases (Explore, Build, Formalize) and can transition between them without closing.

### Cycles
Each cycle follows Plan→Act→Observe→Reflect. Cycles produce observations, and observations accumulate into patterns.

### Patterns
Recurring observations across cycles and lines. When a pattern reaches saturation (Glaser test), it becomes a candidate for theory.

### Theories
Formalized patterns with evidence, conditions of application, and connections. Theories are living documents that evolve.

### The Backlog
Questions waiting to be explored. No limit. Questions can be promoted to lines when capacity allows.

## Operational Rules

1. **Active build limit:** 1 (deep focus)
2. **Active lines limit:** 2-3 (parallel exploration without dispersion)
3. **Golden rule:** Document while doing, not after
4. **Anti-presupposition:** Every new line goes through presupposition detection
5. **Phases, not closures:** When a line produces an artifact, it changes phase — it doesn't close

## Project Structure

```
MAP.md           → System index (auto-generated)
FRAMEWORK.md     → This file (methodology reference)
/intake          → Reference library (original research docs, stay during investigation)
/backlog         → Unexplored questions (unlimited)
/lines           → Active lines of inquiry (max 2-3)
/theories        → Documented emergent theories
/paused          → Temporarily paused lines
/closed          → Closed lines with closure document
/shared          → Reusable artifacts
  /code          → Reusable code
  /patterns      → Documented patterns
  /templates     → Document templates
```
