# Session Context

## Active Work

Línea de investigación `knowledge-pipeline` — 3 ciclos completados evaluando si existe un producto que cubra el flujo ingesta→extracción→clasificación→patrones→consultas para fuentes heterogéneas (bookmarks, web, YouTube, TikTok, Instagram). Conclusión: las etapas 1-2 están resueltas, hay que construir las etapas 3-5.

## Key Decisions

- **Pipeline son 5 etapas, no 6**: la acción siempre es del usuario, no del sistema
- **Supadata** ($5/mo) como servicio de extracción universal (web + YouTube + TikTok + Instagram en un solo API)
- **Jina Reader** (gratis) como fallback para web→markdown
- **Productos descartados por el usuario tras probarlos**: Recall (no genera conexiones automáticas reales), NotebookLM (no transcribe videos, respuestas escuetas), Readwise (solo lectura de libros)
- **Python** como lenguaje para la v1 (ecosistema ML/AI completo)
- **Sin LangGraph inicialmente** — empezar con Python puro, migrar después si crece la complejidad
- **Sin chatbot** — CLI script es suficiente para validar
- **Arquitectura v1**: Supadata → BERTopic (clasificación) → ChromaDB (vector DB) → Ollama/LLM (consultas RAG)
- **Clasificación self-hosted**: BERTopic como opción principal, Ollama+prompt como alternativa simple
